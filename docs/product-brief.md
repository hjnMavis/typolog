# Typolog — Product Brief

## 서비스 개요

- **이름**: 타이포로그 / Typolog (Typography + Log)
- **한 줄 요약**: 같은 문장을, 각자의 일상에서 전혀 다르게 완성하는 글자 콜라주 앱
- **플랫폼**: 모바일 웹 (PWA 고려)
- **언어**: 한국어

사용자는 매일 제공되는 짧은 한국어 문장(공백 제외 6~10글자)을 보고, 자신의 일상 속 간판·전단지·건물·포스터·물건 등에서 글자를 찾아 카메라로 촬영한 뒤, 한 글자씩 crop하여 문장을 완성한다. 완성된 결과물은 ransom note collage / typography collage 느낌의 이미지가 된다.

같은 문장이라도 사람마다 폰트, 색, 질감, 출처가 달라서 결과물이 완전히 다르다.

## 타겟 사용자

| 구분 | 설명 |
|------|------|
| 연령대 | 20대~30대 |
| 성향 | 일상에 소소한 재미를 원하는 사람 |
| 기록 방식 | 직접적인 자랑보다 취향과 동선이 자연스럽게 묻어나는 기록을 선호 |
| 공유 채널 | Instagram Story, X, Threads 등에 공유할 만한 비주얼 결과물을 좋아함 |
| 핵심 동기 | "글자를 모으다 보면 내 동선과 일상이 자연스럽게 드러난다" |

### 페르소나

**지은 (27세, 디자이너)**
- 출퇴근길에 눈에 띄는 간판을 사진 찍는 습관이 있음
- Instagram Story에 감성 사진을 자주 올림
- "오늘 내가 본 글자들"이라는 컨셉에 매력을 느낌
- 완성된 콜라주가 예쁘면 바로 공유하고 싶어함

**민수 (31세, 마케터)**
- 하루 일과 중 카페, 지하철, 식당을 돌아다님
- 같은 문장인데 친구와 결과가 완전히 다른 게 재밌음
- 경쟁보다는 서로 비교하며 웃는 것이 목적

## 핵심 가치

1. **일상 재발견**: 매일 지나치던 글자들을 의식적으로 찾게 됨
2. **자연스러운 자기표현**: 어떤 글자를 어디서 찾았느냐가 그 사람의 동선과 취향을 드러냄
3. **같은 주제, 다른 결과**: 같은 문장이라도 완전히 다른 콜라주가 나옴 → 비교의 재미
4. **공유 가치 있는 결과물**: SNS에 올릴 만한 독특한 비주얼

## MVP 핵심 플로우

```
1. 오늘의 챌린지 문장 확인
   └─ "오늘도 화이팅" (공백 제외 6글자)

2. 글자 슬롯 확인
   └─ [오] [늘] [도] [화] [이] [팅]  (공백 제외)

3. 슬롯 터치
   └─ [오] 슬롯 선택

4. 글자 촬영/업로드
   └─ 카메라로 간판의 "오" 촬영 또는 갤러리에서 선택

5. 글자 crop
   └─ 사진에서 "오" 영역을 직접 드래그하여 crop

6. 슬롯에 저장
   └─ [오✓] [늘] [도] [화] [이] [팅]

7. 반복 (모든 슬롯 채우기)
   └─ [오✓] [늘✓] [도✓] [화✓] [이✓] [팅✓]
   └─ 이미 채운 슬롯을 다시 터치하면 교체 가능

7-1. 이어하기 (중단 후 복귀 시)
   └─ 진행 중인 챌린지가 있으면 "이어하기" 버튼 표시
   └─ 이전에 채운 슬롯 상태가 복원됨

8. 콜라주 미리보기
   └─ 각 글자 조각이 배치된 콜라주 이미지 확인
   └─ 배경색 선택 (흰/검/크림)
   └─ "다시 수정" 버튼으로 글자 수집 화면 복귀 가능

9. 콜라주 완성 & 다운로드
   └─ 콜라주 PNG 이미지 생성
   └─ 기기에 다운로드 가능

10. 제출
    └─ 공개/비공개 선택 후 서버에 제출

11. 공유
    └─ 공유 링크 생성, SNS 공유

12. 피드 감상
    └─ 같은 문장을 완성한 다른 사람들의 결과물 비교, 좋아요
```

## 제외할 기능 (MVP 이후)

| 기능 | 제외 이유 |
|------|----------|
| OCR 자동 글자 인식 | 핵심 가치는 "직접 찾아서 crop하는 행위" 자체. OCR은 보조 기능으로만 나중에 고려 |
| 댓글 | 좋아요만으로 충분한 인터랙션. 댓글은 모더레이션 부담 증가 |
| 팔로우/팔로잉 | 피드가 "문장 중심"이라 사람 중심 소셜 기능은 불필요 |
| 과거 챌린지 아카이브 | 오늘의 문장에 집중. 아카이브는 데이터 쌓인 후 고려 |
| 알림 시스템 (Push) | 모바일 웹 Push는 복잡도 대비 효과 불확실 |
| 관리자 대시보드 UI | SQL + seed로 문장 등록. 관리자 UI는 운영 필요 시 |
| 다국어 지원 | 한국어 한정으로 시작 |
| 콜라주 레이아웃 선택 | 기본 레이아웃 하나로 시작 |
| 원본 사진 저장 | 프라이버시 + 스토리지 절약. crop 이미지만 저장 |
| 랭킹/리더보드 | 경쟁보다 감상이 핵심 |
| 스트릭/연속 달성 보상 | MVP에서는 단순하게 |

## 성공 지표

### 핵심 지표 (Primary Metrics)

| 지표 | 정의 | 목표 (베타 1개월) |
|------|------|------------------|
| 일일 활성 사용자 (DAU) | 하루에 한 번 이상 접속한 사용자 수 | 50+ |
| 콜라주 완성률 | 챌린지 시작 → 콜라주 완성 비율 | 40% 이상 |
| 공유율 | 완성된 콜라주 중 공유 링크 생성 비율 | 20% 이상 |

### 퍼널 지표 (Funnel Metrics)

```
문장 확인 (100%)
  → 챌린지 시작 (목표: 60%)
    → 첫 글자 crop 완료 (목표: 80%)
      → 전체 글자 완성 (목표: 50%)
        → 제출 (목표: 90%)
          → 공유 (목표: 30%)
```

### 보조 지표

| 지표 | 정의 |
|------|------|
| 글자당 평균 crop 시간 | 하나의 글자를 촬영~crop하는데 걸리는 시간 |
| 세션 길이 | 한 번 접속 후 체류 시간 |
| 피드 체류 시간 | 피드에서 다른 사람 결과물을 보는 시간 |
| 좋아요 비율 | 피드 조회 대비 좋아요 클릭 비율 |
| 바이럴 계수 | 공유 링크를 통해 유입된 신규 사용자 비율 |
| D1/D7 리텐션 | 가입 후 1일/7일 후 재방문 비율 |
